#  How to Use Project Templates

<div align="center">

** Quick Start Guide for New Projects**

</div>

##  Creating a New Project

Follow these steps to set up your project documentation:

1.  **Copy the template**
   ```bash
   cp -r /templates/project_template /projects/your-project-name
   ```

2.  **Rename folder** to your project name

3.  **Fill in all required sections** in each markdown file

4.  **Define ownership** and assign responsibilities

5.  **Commit to `/projects`** with proper documentation

>  Tip: Use descriptive project names that reflect the system's purpose.

---

##  Documentation Rules

###  Best Practices

-  **Keep files small and focused** - One topic per file
-  **Avoid large monolithic documents** - Break them down
-  **Update on every significant change** - Keep docs current
-  **Use tables for coverage clarity** - Visual representation helps
-  **Include last update date** - Track freshness

###  Quality Standards

-  **Clear, concise language** - Avoid jargon when possible
-  **Internal links** - Connect related documentation
-  **Consistent formatting** - Follow established patterns


---

##  Update Triggers

Update documentation **immediately** when:

-  **New feature added** - Functional changes
-  **New integration introduced** - External dependencies
-  **Automation coverage changes** - Test updates
-  **Risk profile changes** - Priority or impact shifts
-  **Team changes** - Ownership or role updates
-  **Architecture changes** - System design modifications
